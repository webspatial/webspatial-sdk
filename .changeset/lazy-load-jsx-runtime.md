Internal lazy-load JSX runtime unification + style-mutation bug fix (PR 3 of 6 for `openspec/lazy-load-spatial-runtime`). Updates `packages/react/src/jsx/jsx-shared.ts` so `Model`, `withSpatialized2DElementContainer`, and `withSpatialMonitor` resolve to the facade versions added in PR 2; strip + facade-HOC wrap now runs on every JSX call site (`jsx`, `jsxs`, `jsxDEV`, and the deprecated `createElement`) in a single pass with all markers (`enable-xr`, `enable-xr-monitor`, `style.enableXr`, `__enableXr__` class token) stripped even when combined. Fixes a latent bug where `props.style.enableXr` was deleted via in-place mutation of the user-supplied style object — the runtime now spread-clones `props.style` so memoized / `Object.freeze`d style references stay intact. The `props.class` HTML attribute spelling is explicitly NOT treated as a marker source (only `props.className` is). Adds the `@deprecated` JSDoc to the `createElement` export per the spec's deprecation contract — runtime behavior is unchanged in v1. `jsx-runtime.web.ts` and `jsx-dev-runtime.web.ts` are reduced to pass-through re-exports of the unified runtime as a transitional shim; physical deletion (plus the matching `tsup` and `package.json` exports drops) is queued for PR 5. No public API surface changes in this release.
