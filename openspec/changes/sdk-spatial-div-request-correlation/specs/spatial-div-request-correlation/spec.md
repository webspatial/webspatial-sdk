## ADDED Requirements

### Requirement: SpatialDiv creation requests carry refresh-safe metadata

The SDK SHALL include refresh-safe request metadata when opening platform protocol URLs for `createSpatialized2DElement` and `createAttachment`.

The metadata SHALL include:

- `wsrid`: an opaque request id generated by the frontend SDK.
- `wsepoch`: a page lifecycle epoch value when the host runtime exposes one to the SDK.

#### Scenario: SpatialDiv creation URL includes request id

- **WHEN** the SDK opens a `webspatial://createSpatialized2DElement` protocol URL
- **THEN** the URL SHALL include a non-empty `wsrid` query parameter

#### Scenario: Attachment creation URL includes request id

- **WHEN** the SDK opens a `webspatial://createAttachment` protocol URL
- **THEN** the URL SHALL include a non-empty `wsrid` query parameter

#### Scenario: Host page epoch is available

- **GIVEN** the host runtime exposes a current page epoch to the SDK
- **WHEN** the SDK opens a SpatialDiv or attachment creation protocol URL
- **THEN** the URL SHALL include `wsepoch` with that epoch value

#### Scenario: Host page epoch is unavailable

- **GIVEN** no host page epoch is available to the SDK
- **WHEN** the SDK opens a SpatialDiv or attachment creation protocol URL
- **THEN** the URL SHALL still include a valid `wsrid`

### Requirement: Request ids are opaque and reload-safe

`wsrid` SHALL be treated as an opaque correlation key by host platforms. The SDK MAY generate it using an internal nonce and sequence, but consumers MUST NOT depend on its internal format.

#### Scenario: New JavaScript context creates request ids

- **WHEN** SDK request id generation is initialized after a page reload
- **THEN** newly generated `wsrid` values SHALL not collide with values generated by the previous JavaScript context in practical usage

### Requirement: Pending creation callbacks are cleaned up

The SDK SHALL remove pending request receivers for SpatialDiv and attachment creation after either successful completion or timeout.

#### Scenario: Native creation callback succeeds

- **WHEN** a native creation callback is received for `wsrid`
- **THEN** the SDK SHALL remove the pending receiver for that request

#### Scenario: Native creation callback never arrives

- **WHEN** no native creation callback is received before the configured timeout
- **THEN** the SDK SHALL remove the pending receiver for that request
- **AND** the creation promise SHALL settle with a failure result or equivalent error path
