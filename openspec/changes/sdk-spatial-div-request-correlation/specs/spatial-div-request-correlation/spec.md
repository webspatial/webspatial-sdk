## ADDED Requirements

### Requirement: SDK emits refresh-safe request correlation ids

The frontend SDK SHALL emit a non-empty `rid` for each SpatialDiv and attachment creation request.

#### Scenario: SpatialDiv request is created

- **WHEN** the SDK creates a SpatialDiv request URL
- **THEN** the URL SHALL include a non-empty `rid` query parameter

#### Scenario: Attachment request is created

- **WHEN** the SDK creates an attachment request URL
- **THEN** the URL SHALL include a non-empty `rid` query parameter

#### Scenario: Page reload creates a new JavaScript context

- **WHEN** the SDK generates `rid` values after page reload or module reinitialization
- **THEN** newly generated `rid` values SHALL avoid practical collisions with values generated by the previous JavaScript context

### Requirement: SDK emits page ownership metadata when available

The frontend SDK SHALL emit `wsepoch` when host-provided page epoch metadata is available.

#### Scenario: Host page epoch is available

- **GIVEN** the host exposes page epoch metadata to the SDK
- **WHEN** the SDK creates a SpatialDiv or attachment request
- **THEN** the URL SHALL include `wsepoch` with that epoch value

#### Scenario: Host page epoch is unavailable

- **GIVEN** the host does not expose page epoch metadata to the SDK
- **WHEN** the SDK creates a SpatialDiv or attachment request
- **THEN** the URL MAY omit `wsepoch`
- **AND** SHALL still include a valid `rid`

### Requirement: SDK keeps callback correlation on rid

Frontend completion handling SHALL continue to correlate SpatialDiv creation through `rid`.

#### Scenario: Native creation callback is received

- **WHEN** a native creation callback is received for a request `rid`
- **THEN** the SDK SHALL resolve or reject the matching pending request using that same `rid`

#### Scenario: Host path does not consume wsepoch

- **GIVEN** the host path does not consume `wsepoch`
- **WHEN** the request completes through `rid`
- **THEN** the SDK SHALL remain compatible
- **AND** stale-request rejection behavior depends on host support for `wsepoch`
